# 优化版管线使用说明

## 概述

优化版管线实现了您要求的完整流程：

```
角色信息表并行填表 → 故事主线 → 全书大纲 → 章节大纲 → 章节切片并行写作 → 实时矩阵监测
```

## 核心特性

### 1. 角色信息表并行填表
- 使用固定的角色信息表模板
- AI并行填表生成用户指定数量的角色
- 用户可指定主角和反派名称，AI自动补全其他角色

### 2. 故事主线生成
- 基于主题和已生成的角色信息
- 生成完整的主线剧情（按卷划分阶段）
- 包含核心冲突和结局方向

### 3. 全书大纲生成
- 基于新增的主线内容续写大纲
- 包含卷级结构和主要事件

### 4. 章节大纲规划
- 基于全书大纲规划后续章节大纲
- 为每章分配事件和概要

### 5. 章节切片并行写作
- 每个章节分成固定数量的切片（默认4个）
- 切片类型：开场 → 发展 → 高潮 → 结尾
- 所有章节的切片都可以并行书写

### 6. 章节并行矩阵视图
- 实时监测所有章节切片的生成进度
- 每个章节占用一个矩阵行
- 每行内展示章节对应的切片状态
- 支持自动刷新（每2秒）

## 使用方式

### Web UI方式

1. 启动项目：
   ```bash
   python main.py --web
   ```

2. 访问：http://localhost:5000/create

3. 点击"优化版管线"标签

4. 填写配置：
   - **角色总数**：要生成的角色数量
   - **主角名称**：用户指定的主角（必填）
   - **反派名称**：可选
   - **卷数**：小说总卷数
   - **每卷章节数**：每卷包含的章节数
   - **每章切片数**：每个章节分成的切片数（2-8）
   - **并发上限**：最大并发任务数

5. 点击"⚡ 启动优化版管线"

6. 自动打开矩阵视图页面，实时查看进度

### 矩阵视图

访问：http://localhost:5000/matrix

矩阵视图显示：
- 管线状态（空闲/运行中/已完成）
- 当前阶段
- 总任务数/已完成数
- 进度百分比
- 章节矩阵表格：
  - 每行代表一个章节
  - 每列代表一个切片
  - 单元格显示切片状态（等待中/生成中/已完成/错误）
  - 每行末尾显示该章节的完成进度条

## 输出文件

管线执行完成后，输出文件位于 `output/` 目录：

```
output/
├── characters.json      # 角色信息表
├── main_storyline.json  # 故事主线
├── outline.json         # 全书大纲
├── chapters.jsonl       # 章节大纲
└── draft/               # 章节正文
    ├── 0001.md
    ├── 0002.md
    └── ...
```

## 技术架构

### 核心模块

- **角色信息表生成器**：`src/core/character_table_generator.py`
  - 使用固定模板
  - 并行填表生成角色

- **优化版管线编排器**：`src/core/optimized_pipeline.py`
  - 完整的6阶段管线
  - 章节切片并行写作
  - 实时进度监测

- **矩阵视图页面**：`src/web/templates/matrix.html`
  - 实时显示章节矩阵
  - 自动刷新功能

- **Web API**：`src/web/app.py`
  - `/api/pipeline/optimized` - 启动优化版管线
  - `/api/pipeline/progress` - 获取实时进度
  - `/matrix` - 矩阵视图页面

## 切片类型说明

每个章节默认分为4个切片：

1. **开场**：场景设定和角色引入
2. **发展**：情节推进和冲突升级
3. **高潮**：冲突爆发和关键转折
4. **结尾**：冲突解决和伏笔埋设

## 性能优化

### 并发上限设置
- **低配置**（1-16）：适合本地测试
- **中配置**（17-96）：适合常规使用
- **高配置**（97-960）：需要强大的计算资源

### 切片数量设置
- **2切片**：快速生成，适合测试
- **4切片**：平衡质量和速度（推荐）
- **6-8切片**：更精细的控制，适合长章节

## 注意事项

1. 确保RWKV推理服务已启动
2. 大模型推理需要时间，请耐心等待
3. 并发数越高，资源消耗越大
4. 矩阵视图支持自动刷新，可实时查看进度
